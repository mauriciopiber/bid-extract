# Session 2 — 2026-03-26

## What we built

- **PostgreSQL + Drizzle schema**: layouts, prompts, extractions, evals, contests, prompt_evolutions, run_logs, page_types
- **Action layer**: `createAction` pattern — one definition works from CLI, MCP, API, web
- **DB-backed pipeline**: every step logged to run_logs, every extraction scored (math + completeness), everything in postgres
- **Per-page classification**: classifier runs on each page individually, not just first 2 pages
- **Dynamic page types from DB**: page types stored in `page_types` table, classifier prompt builds dynamically
- **Human approval gates**: new page types start as "pending", only "approved" types used in classifier prompt
- **Layout-based UI**: home page shows layouts grouped by fingerprint, click through to extractions

## Key decisions

### 1. DB is source of truth, disk is debug dump
Pipeline writes everything to postgres. CLI `--dump` flag for disk output is optional debug only.

### 2. createAction pattern
Same handler runs from CLI, API, MCP. Actions: extract, stats, extractions, show, dump.

### 3. Per-page classification with page types
- bid_tabulation: line items with quantities, prices
- bid_ranking: just names + totals (NO items — and that's correct, not an error)
- cover, summary, other

Page types are dynamic — stored in DB, not hardcoded. Classifier only uses approved types.

### 4. Human approves all new concepts
LLM proposes, human confirms. Applies to:
- New page types
- New layouts
- Format reclassifications
- Prompt promotions

Why: LLMs hallucinate categories. A slightly different bid tabulation is NOT a new type.

### 5. Type-specific validation
- bid_ranking with 0 line items = CORRECT
- bid_tabulation with 0 line items = ERROR
- Each page type defines what "valid" and "complete" means

## What we learned

### closeDb() fixes hanging CLI
The postgres connection keeps Node alive. Adding `closeDb()` in Commander's `postAction` hook fixes it. Commands now exit instantly.

### Schema duplication for UI
Next.js can't import from `../src/db/schema.ts` — different module boundary. Had to copy schema to `ui/src/lib/schema.ts`. Need a better solution (shared package or monorepo).

### Format type derivation
Document-level format type is derived from page-level classifications:
- Has bid_tabulation pages with 4+ bidders → multi-bidder-matrix
- Has bid_tabulation pages with unit prices → engineering-firm
- Only bid_ranking pages → summary-only
- Has handwriting → handwritten
