# Session 3 — 2026-03-26

## The Big Insight: We Were Doing It Wrong

We were sending full documents or full pages as single blobs. The right approach:
- **Send chunks** — one section, one bid group, or even one portion of a page at a time
- **Pass context forward** — bidder names from page 1 carry to page 2+
- **Never send everything at once** — the more data, the more errors, the harder to verify

## Schema That Holds

```
Document
  project: ProjectInfo
  contracts: Contract[]          ← 1 to N (most have 1)
    name: "Contract 1" | "Schedule A"
    bidGroups: BidGroup[]        ← Base Bid, Supplemental, Alternates
      type: "base" | "supplemental" | "alternate"
      name: string
      sections: Section[]        ← optional grouping
        name: "Bridge Items" | "Roadway Items" | etc
        items: Item[]
          itemNo, description, unit, quantity
          subItems?: Item[]      ← nested (1a, 1b under 1)
          bids: { [bidderName]: { unitPrice, extendedPrice } }
          engineerEstimate?: { unitPrice, extendedPrice }
        subtotal?: { [bidderName]: number }
      total: { [bidderName]: number }
  bidders: BidderInfo[]          ← names, ranks, grand totals
  engineerEstimate?: { total }
```

Key findings:
- **Contract = Schedule** — same concept, different names
- **Bid Group** = Base Bid, Supplemental, or Alternate — within a contract
- **Section** = grouping within a bid group (Bridge Items, Roadway Items) — optional
- **Sub-items** = 1a, 1b, 1c under item 1 — optional
- **Allowances** = separate from alternates (KC docs) — needs handling
- **Bid ranking** = completely separate document type, just names + totals

## Extraction Strategy: Chunks Not Blobs

**NEVER send the full document at once.** Send chunks:

1. **First pass**: Classify all pages. Identify bidder names, contracts, sections.
2. **Per section/bid group**: Extract one section at a time with context (bidder names, contract name)
3. **Context carries forward**: Page 1 establishes bidders → pages 2+ use those names
4. **Merge is trivial**: Same bidder name = same bidder. Items concatenate.

Why chunks:
- Smaller input = more accurate output
- Easier to verify (reviewer sees one section at a time)
- Failed extraction only affects one chunk, not the whole document
- Can retry individual chunks without re-processing everything

## Context Passing (proven working)

Page 1 extracted 3 bidders: "A & B Equipment, LLC", "Gents Enterprises, Inc", "Martin General Contractors"

Pages 2-4 received: "The bidders are: [list]. Extract data for THESE SAME bidders."

Result: 3 clean bidders instead of 10 fragmented ones. The merge worked because same name = same bidder.

## What Failed

- **Keyword-based categories** — regex grouping of items into "Concrete", "Steel" etc. Completely wrong. Removed.
- **Single-pass full-document extraction** — loses data on multi-page PDFs
- **Page-by-page extraction WITHOUT context** — fragments bidders (same bidder on different pages = separate entries)
- **Fake "clean" scores** — 0 line items = 0 warnings = looks clean. Need completeness checks.
