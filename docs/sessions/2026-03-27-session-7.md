# Session 7 — 2026-03-27

## Breakthroughs

### generateObject + flat Zod schema = reliable extraction
- Vercel AI SDK `generateObject` eliminates JSON parse errors entirely
- Flat schema (items with sectionName) works better than nested (contracts/bidGroups/sections/items)
- Claude returns malformed nested structures but handles flat arrays perfectly
- `PageExtractionSchema` in `src/schemas/zod.ts` is THE extraction schema

### .describe() on Zod fields guides the LLM
- Adding `.describe("Engineer's estimate total from the Total row")` to schema fields makes `generateObject` fill them correctly
- This is how you tell the LLM what each field means — through the schema, not just the prompt

### Eval loop works: PR1 86% → PR2 98% → PR3 100%
- PR1: basic prompt, engineer estimate treated as bidder, no totals
- PR2: added "engineer estimate is NOT a bidder", bidder totals instruction
- PR3: explicit total row instruction, lump sum detection rules
- Each version fixed specific failures found in eval comparison

### Lump sum detection
- `isLumpSum: true` when unitPrice = extendedPrice regardless of quantity
- Validator must check: lump sum → unitPrice = extendedPrice, non-lump-sum → unitPrice × qty = extendedPrice
- The LLM sometimes divides extended/qty to get a fake unit price — prompt must explicitly say "NEVER divide"

## Architecture Decisions

### One source of truth for types
- Zod schemas in `src/schemas/zod.ts` — everything imports from here
- TypeScript types inferred with `z.infer<>` — no manual interfaces
- NEVER define schemas inline in scripts

### Flat ↔ Hierarchical conversion
- `toHierarchical()` and `toFlat()` in `src/schemas/convert.ts`
- 1:1 lossless roundtrip — 8 tests prove it
- LLM produces flat → convert to hierarchical immediately → everything downstream uses hierarchical
- Key: `sectionSubtotals` field on flat format maps to `section.subtotals` on hierarchical

### Eval framework (files, not DB)
- References: `evals/reference/S01.json` — human-verified ground truth
- Runs: `evals/results/S02/E1/PR3/run-001.json` — extraction output
- Scripts: `run.ts`, `compare.ts`, `generate-reference.ts`, `review.ts`, `verify.ts`, `check-math.ts`, `prepare-samples.ts`
- Git-tracked — diffs show exactly what changed between prompt versions

### Action words
- "human review" → open image + JSON, wait for feedback
- "verify" → mark reference as verified
- "run eval" → run extraction + compare
- "check math" → validate unitPrice × qty = extendedPrice
