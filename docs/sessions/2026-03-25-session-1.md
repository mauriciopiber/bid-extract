## Learnings

### 1. Lump Sum confusion (SOLVED)
**Problem**: Model confused "Approx Qty" with unit prices. Fabricated unitPrice by dividing.
**Fix**: Explicit prompt rules. "NEVER back-calculate unitPrice."

### 2. Character-level vision misreads (PARTIALLY SOLVED)
**Problem**: `273.48` read as `273.46`.
**Fix**: Math Resolver — deterministic, trust hierarchy: qty > extended > unitPrice.

### 3. Total mismatch $24K (PARTIALLY SOLVED)
**Problem**: Louis-Company sum off by $24K. Contest found $27,156 should be $2,715.
**Status**: Need re-extraction to verify.

### 4. JSON parse failures (SOLVED)
**Fix**: Robust parser + retry logic.

### 5. NEVER guess or infer structure (LESSON LEARNED)
**Problem**: Added keyword-based category grouping. Completely wrong.
**Fix**: Removed. Rule: only show what the document explicitly contains.

### 6. Missing line items on multi-bidder-matrix (FOUND, UNSOLVED)
**Problem**: Barry Co has 4 bidders × 21 items in a wide matrix. Got names + totals but **0 line items**.
**Impact**: 78% clean rate is inflated. Real accuracy is lower.
