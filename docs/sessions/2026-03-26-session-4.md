# Session 4 — 2026-03-26

## The Problem: Regression

We keep breaking things that were working. Examples:
- Bid ranking extraction worked → schema change broke it
- Bidder totals showed up → refactor lost them
- UI was loading → code change crashed it

Every "improvement" introduces regression because there are no guardrails.

## The Fix: Snapshot Tests

Every extraction that a human verifies as correct becomes a frozen test case.

1. Human reviews extraction in UI → says "this is correct"
2. System saves a snapshot: PDF file + expected output
3. Before ANY code change, run all snapshots
4. If any snapshot output changes → REJECT the change

This is NOT promptfoo evals. This is simple: same input → same output. If it changes, something regressed.

## What we need

```
test_cases/
  bollinger_co_ranking.json     ← expected output (human verified)
  andrew_bridge_tabulation.json ← expected output
  ...
```

CLI command:
```
pnpm cli test:snapshots   ← runs all PDFs through pipeline, compares to expected
```

Rule for CLAUDE.md:
- **NEVER merge code that breaks a snapshot test**
- **Every human-verified extraction becomes a snapshot**
- **Snapshots are immutable — only human can update them**
